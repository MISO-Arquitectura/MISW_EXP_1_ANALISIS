Anlisis de resultados

El experimento consta de desplegar 3 instancias de un microservicio que ante una suma de dos operandos y un resultado debe contestar si el resultado es correcto o no (es decir calificar una pregunta). Cada microservicio tiene una probabilidad de falla. Realizaremos 3 simulaciones con 3 probabilidades distintas de falla as:

  1. prob_falla = 1%
  2. prob_falla = 0.5%
  3. prob_falla = 5%

En cada simulacin se correr un ciclo con 20,000 iteraciones en donde en cada iteracin el componente validador se encargar de enviar una peticin a los 3 microservicios, recibir las respuestas (True o False) y compararlas. Si existe discrepancia en las respuestas, el componente validador determinar que el microservicio que contesta diferente a los otros dos presenta una falla (en esta tctica lo que conteste la mayora se asume como la respuesta correcta).

Calificamos la veracidad del validador

A continuacin vamos a etiquetar cada fila individual de nuestros datos as:

  • true_positive: Corresponde a aquel caso en el que nuestro validador detect correctamente cuando un microservicio estaba fallando. Por ejemplo ante la consulta 2 + 2 = 4, dos de los microservicios contestaron que era correcta, uno contest que era incorrecta y por consiguiente el validador detect correctamente que el que contest diferente estaba fallando.
  • true_negative: Corresponde a aquel caso en el que nuestro validador detect que todos nuestros microservicios estaban funcionando correctamente. Por ejemplo ante la consulta 2 + 2 = 4, los tres microservicios contestaron que era correcta y el validador detect que ninguno fallaba.
  • false_positive: Corresponde a aquel caso en el que nuestro validador detect que un microservicio estaba fallando y en realidad estaba funcionando correctamente. Por ejemplo ante la consulta 2 + 2 = 4, dos de los microservicios contestaron que era incorrecta, uno contest que era correcta y por consiguiente el validador detect errneamente que el que contest diferente estaba fallando.
  • false_negative: Corresponde a aquel caso en el que nuestro validador no detect que un microservicio estaba fallando. Por ejemplo ante la consulta 2 + 2 = 4, los 3 microservicios contestaron que era incorrecta y por consiguiente el validador detect errneamente que no exista falla alguna.

Es de aclarar que se podra argumentar que un falso positivo tambin implica un falso negativo; por ejemplo, si ante la consulta (2 + 2) == 4, dos microservicios contestan que es incorrecta y uno contesta que es correcta, el validador habr identificado errneamente (falso positivo) que el servicio que contest diferente est fallando cuando no lo est, y a su vez no habra detectado que los que contestaron de forma incorrecta estaban fallando (falsos negativos). Sin embargo, este caso lo vamos a etiquetar como falso positivo para simplificar el anlisis.

def label_validator(row):
    server_responses = {
        1: row['server_1_response'],
        2: row['server_2_response'],
        3: row['server_3_response']
    }

    expected_server_response = row['expected_server_response']
    validator_unavailability_resolution = row['unavailable_server']

    if validator_unavailability_resolution == 'None':
        if all(res == expected_server_response for res in server_responses.values()):
            return 'true_negative'
        elif all(res != expected_server_response for res in server_responses.values()):
            return 'false_negative'
    elif server_responses[validator_unavailability_resolution] == expected_server_response:
        return 'false_positive'
    elif server_responses[validator_unavailability_resolution] != expected_server_response:
        return 'true_positive'

Conclusiones del experimento

Como pudimos apreciar, el porcentaje de fallas que somos capaces de detectar correctamente con un componente validador y 3 instancias de un mismo servicio pareciera estar directamente relacionado con la probabilidad de falla de un componente. Nuestra hiptesis de diseo estipula que somos capaces de detectar las fallas correctamente el 99% de las veces con un componente validador y 3 instancias del mismo servicio. En el experimento 1 (con una probabilidad de falla de un componente igual al 1%) obtuvimos un resultado igual a el 98.76% de fallas detectadas correctamente, que s lo redondeamos obtenemos el 99%, con lo cual se validara nuestra hiptesis de diseo; sin embargo, si el porcentaje de falla de un componente es igual al 5% como en el experimento 3, el resultado nos dice que no somos capaces de detectar el 99% de las fallas correctamente, ya que obtuvimos el 95.22% como resultado.

Los resultados obtenidos nos invitan a realizar un anlisis ms profundo utilizando como base la teora de probabilidad. En este anlisis revisaremos para varios valores de probabilidad de falla de un componente y cantidades (impares) diferentes de servicios, si somos capaces de derivar una funcin que relacione el porcentaje de fallas detectadas correctamente por un validador con estas dos variables (nmero de servicios y probabilidad de falla de un servicio).

Anlisis de probabilidad

Revisemos un poco de teora de probabilidad y corroboremos si los resultados obtenidos por el experimento estn acordes a la teora.

Definimos,

$P_1 = 0.01$ como la probabilidad de que el servidor 1 falle

$P_2 = 0.01$ como la probabilidad de que el servidor 2 falle

$P_3 = 0.01$ como la probabilidad de que el servidor 3 falle

As mismo definimos,

$1-P_1 = 0.99$ como la probabilidad de que el servidor 1 no falle

$1-P_2 = 0.99$ como la probabilidad de que el servidor 2 no falle

$1-P_3 = 0.99$ como la probabilidad de que el servidor 3 no falle

Definimos tambin la probabilidad de que al menos uno de los componentes falle (escenarios 2 a 8) como,

$P_f = 1-(1-P_1)*(1-P_2)*(1-P_3)$

Es de aclarar que las probabilidades de ocurrencia de los 3 eventos son independientes entre s (que el servicio 1 falle no afecta que falle el servicio 2 o que falle el servicio 3).

Para derivar la ecuacin anterior, recordemos que la suma de las probabilidades de ocurrencia de todos los escenarios o eventos posibles es igual a 1. As mismo sabemos que la probabilidad de ocurrencia del escenario 1 es igual a $(1-P_1)*(1-P_2)*(1-P_3)$ (Evento en el que ninguno de los componentes falla). Por consiguiente si a 1 le restamos la probabilidad de ocurrencia del escenario 1 habremos obtenido la probabilidad de que ocurra alguno de los escenarios 2 a 8 (Por lo menos un componente falla)

Podemos ir actualizando nuestra tabla de escenarios as:

escenario server_1 server_2 server_3 falla_segn_validador label prob
1 no falla no falla no falla ninguno true_negative $(1-P_1)*(1-P_2)*(1-P_3)$
2 no falla no falla falla server_3 true_positive $(1-P_1)*(1-P_2)*P_3$
3 no falla falla no falla server_2 true_positive $(1-P_1)*P_2*(1-P_3)$
4 falla no falla no falla server_1 true_positive $P_1*(1-P_2)*(1-P_3)$
5 falla falla no falla server_3 false_positive $P_1*P_2*(1-P_3)$
6 falla no falla falla server_2 false_positive $P_1*(1-P_2)*P_3$
7 no falla falla falla server_1 false_positive $(1-P_1)*P_2*P_3$
8 falla falla falla ninguno false_negative $P_1*P_2*P_3$
Total $$\sum_{\text{escenario} = 1}^{8}{P_{escenario}} = 1$$

Ahora nos interesa calcular la probabilidad de tener un true_positive dado que por lo menos uno de los componentes falla (Esto es la probabilidad de que en caso de una falla el componente validador sea capaz de detectarla correctamente). Lo anterior lo calculamos as:

$$ proportion = P(\text{true\_positive} \mid \text{falla}) = \frac{P(\text{true\_positive} \cap \text{falla})}{P_f} $$

Ntese como $P(\text{true\_positive} \cap \text{falla})$ es justamente la suma de probabilidades de los escenarios 3, 4 y 5, entonces: $$ proportion = P(\text{true\_positive} \mid \text{falla}) = \frac{(1-P_1)*(1-P_2)*P_3 + (1-P_1)*P_2*(1-P_3) + P_1*(1-P_2)*(1-P_3) }{P_f} $$

$$ proportion = P(\text{true\_positive} \mid \text{falla}) = \frac{(0.99)*(0.99)*(0.01)*3}{1-(0.99)*(0.99)*(0.99)} $$

proportion = (0.99*0.99*0.01*3)/(1-0.99*0.99*0.99)

Con esto demostramos que los resultados de nuestro experimento (98.76%) son acordes a la teora de probabilidad (98.99%)

Teora de probabilidad aplicada a 5 y 7 microservicios

Revisemos ahora si podemos trasladar el anterior anlisis a 5 o 7 microservicios. Para 3 microservicios vimos que se pueden presentar 8 escenarios. Lo anterior resulta de la ecuacin de permutaciones, que no es otra cosa que el nmero de posibilidades n (falla no falla) que se tienen para un nmero de elementos k (# de servidores):

Para 3 servicios: $$ n^k = 2^3 = 8 $$

Para 5 servicios: $$ n^k = 2^5 = 32 $$

Para 7 servicios: $$ n^k = 2^7 = 128 $$

La tabla para 8 escenarios fue sencilla de construir, pero para 32 o 128 escenarios es una tarea dificil; sin embargo, an vamos a ser capaces de calcular la proporcin de fallas detectadas correctamenre. Veamos como.

Lo anterior debido a que si la minora falla, el componente validador detecta una inconsistencia en las respuestas y toma la respuesta de la mayora como la respuesta correcta, etiquetando as a la minora como servicios que presentan fallas. Para 3 componentes tenemos detecciones de fallas correctas cuando 1 componente fallaba y detecciones de fallas incorrectas cuando 2 o 3 componentes fallaban.

Hagamos las siguientea deficiones:

$P_i = 0.01$ (Probabilidad de falla de 1 servicio)

$(1-P_i) = 0.99$ (Probabilidad de no falla de 1 servicio)

$P_{nf} = 1 - (1-P_i)^n$ (Probabiliad de que en n servicios falle al menos 1)

Ahora vamos a apoyarnos en la frmula de combinaciones, que no es otra cosa que la forma en que ese pueden seleccionar k elementos (nmero de servidores fallando) de un conjunto de n elementos (nmero total de servidores) sin tener en cuenta el rden.

$$ C(n,k) = \binom{n}{k} = \frac{n!}{k!(n-k)!} $$

from math import factorial
def get_percentage_of_detected_failures(num_servers: int, fail_prob: float) -> float:
    if num_servers % 2 == 0:
        return
    no_fail_prob = 1 - fail_prob
    at_least_1_fail_prob =  1 - (no_fail_prob) ** num_servers
    # calculate the number of servers that need to fail in order for the validator to label a true_positive
    failing_servers_for_true_positive = num_servers // 2
    # calculate P(true_positive and failure)
    p_true_positive = 0
    for i in range(1, failing_servers_for_true_positive + 1):
        comb = factorial(num_servers)/(factorial(i)*factorial(num_servers-i))
        p_true_positive += comb * no_fail_prob ** (num_servers - i) * fail_prob ** i
    return p_true_positive / at_least_1_fail_prob

Vimos como la hiptesis se cumple si la probabilidad de falla es del 1%, pero no se cumple si la probabilidad de falla es del 5%; es decir, entre ms propenso a fallar es un servicio menos porcentaje de fallas somos capaces de detectar correctamente con un componente validador.

Al descubrir esta relacin quisimos ahondar ms y revisar si el nmero de servicios instanciados tambin influye en la veracidad o eficiencia del validador (porcentaje de fallas detectadas correctamente). Para lo anterior no hicimos otro experimento de 20,000 peticiones y 5 o 7 servicios con diferentes probabilidades de falla, sino que con un poco de ayuda de teora de probabilidad derivamos una funcin que relaciona el porcentaje de fallas detectadas correctamente con estas 2 variables (nmero de instancias o servicios y probabilidad de falla de un servicio).
